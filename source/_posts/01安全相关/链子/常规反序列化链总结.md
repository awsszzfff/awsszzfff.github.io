## 二、详细分析

### 2.1 CC1 链

#### 两种变体
```java
// 变体1：TransformedMap
Map outerMap = TransformedMap.decorate(innerMap, null, transformerChain);
// 触发点：MapEntry.setValue()

// 变体2：LazyMap
Map lazyMap = LazyMap.decorate(innerMap, transformerChain);
// 触发点：动态代理 + LazyMap.get()
```

#### 关键接口/方法

| 组件 | 关键方法 | 可序列化 | 作用 |
|------|----------|----------|------|
| `AnnotationInvocationHandler` | `readObject()` | ✅ | 反序列化入口 |
| `TransformedMap` | `checkSetValue()` | ✅ | 触发 transform |
| `LazyMap` | `get()` | ✅ | 触发 transform |
| `InvokerTransformer` | `transform()` | ✅ | 反射执行命令 |
| `ConstantTransformer` | `transform()` | ✅ | 返回常量 |
| `ChainedTransformer` | `transform()` | ✅ | 串联执行 |

#### 利用链
```
readObject() 
  → AnnotationInvocationHandler.readObject()
    → memberValue.setValue() / proxy.invoke()
      → TransformedMap.checkSetValue() / LazyMap.get()
        → ChainedTransformer.transform()
          → ConstantTransformer.transform()
          → InvokerTransformer.transform() (反射Runtime)
```

#### 优缺点
**优点**：
- 经典链，原理清晰
- 两种变体适应不同场景

**缺点**：
- 依赖动态代理（LazyMap 变体）
- `InvokerTransformer` 在高版本被限制
- 需要 JDK 内部类

### 2.2 CC2 链

#### 关键接口/方法

| 组件 | 关键方法 | 可序列化 | 作用 |
|------|----------|----------|------|
| `PriorityQueue` | `readObject()` | ✅ | 反序列化入口 |
| `TransformingComparator` | `compare()` | ✅ | 触发 transform |
| `InvokerTransformer` | `transform()` | ✅ | 反射执行命令 |
| `TemplatesImpl` | `newTransformer()` | ✅ | 加载恶意类 |

#### 利用链
```
readObject()
  → PriorityQueue.readObject()
    → heapify()
      → siftDown()
        → siftDownUsingComparator()
          → TransformingComparator.compare()
            → InvokerTransformer.transform()
              → TemplatesImpl.newTransformer()
                → getTransletInstance()
                  → defineTransletClasses()
                    → newInstance() (恶意类)
                      → Runtime.exec()
```

#### 优缺点
**优点**：
- 无需动态代理
- 利用 PriorityQueue 原生类
- 使用 TemplatesImpl 执行任意代码

**缺点**：
- 仍使用 `InvokerTransformer`
- 依赖 CC 4.x 特定版本

## 六、优缺点总结

### 6.1 按技术特点分类

#### A. 基于反射执行的链（CC1、CC2、CC5、CC6、CC7）
**优点**：
- 直接调用 `Runtime.exec()`，简单明了
- 不依赖 TemplatesImpl

**缺点**：
- 依赖 `InvokerTransformer`
- 高版本 JDK 受限

#### B. 基于 TemplatesImpl 的链（CC2、CC3、CC4、CB1）
**优点**：
- 可执行任意字节码
- 绕过 `InvokerTransformer` 限制

**缺点**：
- 需要动态生成字节码
- 依赖特定 JDK 类

#### C. 基于动态代理的链（CC1、CC3）
**优点**：
- 触发方式灵活

**缺点**：
- 依赖 `AnnotationInvocationHandler`
- 构造复杂

#### D. 基于容器的链（CC2、CC4、CC5、CC6、CC7）
**优点**：
- 使用 JDK 原生类
- 通用性强

**缺点**：
- 需要处理占位数据

### 6.2 综合对比

| 维度 | 最佳链 | 原因 |
|------|--------|------|
| **兼容性** | CC6、CB1 | 高版本 JDK 可用，依赖少 |
| **稳定性** | CC6 | 构造简单，触发可靠 |
| **绕过能力** | CC3、CC4、CB1 | 绕过 InvokerTransformer 限制 |
| **简洁性** | CC5、CC6 | 代码量少，逻辑清晰 |
| **通用性** | CB1 | 不依赖 CC 库，使用 Beanutils |

### 7.2 代码层面
```java
// 1. 使用白名单限制反序列化
ObjectInputStream ois = new ObjectInputStream(bais);
ois.setObjectInputFilter(filterInfo -> {
    Class<?> clazz = filterInfo.serialClass();
    if (clazz != null) {
        if (clazz == String.class || clazz == Integer.class) {
            return ObjectInputFilter.Status.ALLOWED;
        }
        return ObjectInputFilter.Status.REJECTED;
    }
    return ObjectInputFilter.Status.UNDECIDED;
});

// 2. 使用安全的序列化框架
// 推荐使用 Jackson、Fastjson 等 JSON 序列化替代 Java 原生序列化

// 3. 启用安全管理器
System.setSecurityManager(new SecurityManager());

// 4. 配置反序列化过滤器（JDK9+）
System.setProperty("jdk.serialFilter", 
    "!org.apache.commons.collections.*;!com.sun.org.apache.xalan.*");
```

## 八、总结

### 8.1 核心要点

1. **所有链的最终目标**：`Runtime.exec()` 或 `TemplatesImpl.newTransformer()`

2. **关键可序列化组件**：
   - 入口类：`AnnotationInvocationHandler`、`PriorityQueue`、`HashSet` 等
   - 桥梁类：`TransformedMap`、`LazyMap`、`TiedMapEntry` 等
   - 执行类：`InvokerTransformer`、`InstantiateTransformer`、`BeanComparator` 等

3. **触发机制**：
   - 比较类方法：`compare()`
   - 对象方法：`toString()`、`hashCode()`、`equals()`
   - 集合方法：`get()`、`setValue()`

4. **演化趋势**：
   - 从特定类到通用容器
   - 从反射执行到类加载执行
   - 从单一库到多库组合
   - 从复杂构造到简洁流程
